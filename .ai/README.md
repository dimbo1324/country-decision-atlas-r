# AI Assistant Rule Modules

Single source of truth for the rules that govern AI assistants (Claude Code,
Codex, and any future agent) working in this repository.

## How it works

```text
.ai/universal/*.md   Portable rules. Valid for ANY project. Never mention
                     project names, paths, or stack specifics.
.ai/project/*.md     Rules specific to THIS project: repo map, commands,
                     domain guardrails, progress tracking.
CLAUDE.md            Thin entry point for Claude Code. Imports every module
                     via native @path syntax.
AGENTS.md            Entry point for Codex. GENERATED by concatenating the
                     same modules (Codex has no import mechanism). Do not
                     edit by hand.
```

Modules are loaded in filename order: `universal/01..NN` first, then
`project/10..NN`. Later content overrides earlier content, so project rules
take precedence over universal rules when they conflict (e.g. a stricter
file-size limit).

## Editing rules

1. Edit or add a module under `.ai/universal/` or `.ai/project/`.
2. Regenerate the Codex entry: `python dev_tools_scripts_runner.py sync-agents`.
3. Commit the module and the regenerated `AGENTS.md` together.

Never edit `AGENTS.md` directly — it carries a generated-file banner and any
manual edit will be lost on the next sync. `CLAUDE.md` needs no regeneration;
Claude Code resolves `@` imports at session start.

## Porting to another project

1. Copy `.ai/universal/` unchanged.
2. Copy `CLAUDE.md` and the sync script
   (`scripts/dev_tools/sync_agents_md.py` or equivalent); adjust import paths
   if the project layout differs.
3. Rewrite everything under `.ai/project/` for the new project.
4. Run the sync script to produce the new `AGENTS.md`.

Keep universal modules truly universal: if a rule needs a project path or a
project command to make sense, it belongs in `.ai/project/`.
