## Этап 4 — многоязычный корпус и устойчивость Unicode

- [+] Библиотека исходных текстовых блоков для всех 15 локалей раздела 8.3
      (`docs/synthetic_data/input_data/locale_corpus.json`).
- [+] Каждый блок привязан к нейтральному семантическому ключу факта
      (страна/риск/сильная сторона), смысл не меняется между локалями.
- [+] Локализованные имена, заголовки, комментарии, числа, даты,
      синтетическая маркировка (`fictional_notice`) на каждую локаль.
- [+] Явная поддержка RTL для `ar-SA`/`fa-IR`: bidi-переупорядочивание +
      арабское reshaping (`python-bidi` + `arabic-reshaper`) для смешанных
      строк (число/URL-заглушка/латиница).
- [+] Набор шрифтов с покрытием нужных письменностей, настроен для
      PDF/DOCX/растровых non-copyable фрагментов (`core/font_registry.py`).
- [-] Отдельный fake-режим теста нейросетевого перевода — НЕ реализован в
      этой задаче; честно отмечено как отступление в техзадании (раздел 23,
      Этап 4) и оставлено для отдельной, согласуемой задачи интеграционного
      транша.

**Проверка Этапа 4.**

- [+] Каждая из 15 локалей создаёт хотя бы один полноценный текстовый
      артефакт.
- [+] Тесты подтверждают сохранение UTF-8/диакритики/кириллицы/
      иероглифов/индийских письменностей/RTL после генерации мира.
- [+] Визуальный рендер (там, где применимо) без квадратов/обрезаний/
      неверного направления — подтверждено вручную (PDF-скан всех 9
      уникальных письменностей в одном файле) и через glyph-coverage тест.
- [+] Поиск/сравнение строк учитывает Unicode-нормализацию — юнит-тест на
      паре NFC/NFD для вьетнамского текста; полнотекстовый поиск по
      синтетическому корпусу вне рамок Этапов 4-5 (нет такого индекса).
- [-] Переводческие артефакты (fake-режим) — не реализовано, см. выше.

## Этап 5 — рендереры документов, manifest и пакет данных

- [+] JSON/XLSX/DOCX/PDF мирового пайплайна читают только канонический
      `SyntheticWorld` и рецепты документов (не независимый рандом) —
      реализовано как параллельный пакет `core/document_rendering/`
      (см. «Отступления» в техзадании: legacy `RandomContentFactory`-
      генераторы намеренно не тронуты).
- [+] Добавлен генератор TXT.
- [+] Режимы copyable/non-copyable/mixed сохранены для PDF/DOCX с
      обязательной маркировкой на каждой странице/во всех форматах.
- [+] Выходные папки организованы по `dataset_id`/локали/типу артефакта
      (раздел 10) — `documents/<locale>/<format>/[<mode>]/`.
- [+] `manifest.json`, отчёт проверки, контрольные суммы, ZIP-пакет,
      `generation_summary.md`.
- [+] Команда `render --dataset <id>` — повторный рендеринг документов из
      уже существующего canonical JSON без повторной генерации мира.

**Проверка Этапа 5.**

- [+] Один факт о стране одинаково отображается в JSON, таблице, TXT и PDF.
- [+] PDF/DOCX/XLSX/TXT открываются штатными библиотеками и проходят
      структурные проверки (включая RTL/CJK-локали через засабсеченные
      шрифты) — `core/artifact_validation.py`, запускается при упаковке.
- [+] Copyable/non-copyable/mixed имеют заявленные свойства при извлечении
      текста (проверено тестами и вручную через pypdf/python-docx).
- [+] Manifest перечисляет каждый файл, хеш, локаль, формат, связанные
      сущности.
- [+] Повторный рендер не меняет канонический мир (`render` только читает
      `canonical/synthetic_world.json`, не запускает `WorldGenerator`).
