## Найденные пробелы относительно буквального текста ТЗ

- [+] Раздел 6.1/17/23: канонические модели должны быть
      «JSON Schema/Pydantic-модели» — сейчас обычные `@dataclass`, без
      JSON Schema и без полей `supported_locales`/`fictional_notice` в
      самих метаданных (сейчас `fictional_notice` только синтезируется в
      `to_dict()`).
- [+] Раздел 23, Этап 1, п.6: «проверка отсутствия реальных персональных
      данных, production-настроек и запрещённых внешних вызовов во входных
      данных» — сейчас проверяются только запрещённые названия стран,
      общей проверки на PII/секреты/URL нет.
- [+] Раздел 23, Этап 1, критерий: «одинаковые конфигурация и seed дают
      одинаковый результат после исключения технического времени создания»
      — текущий тест сравнивает `to_dict()` целиком, включая `generated_on`
      (`date.today()`), совпадение сейчас держится на том, что оба вызова
      происходят в один день, а не на явном исключении поля.
- [+] Раздел 7.2/14.2: «отрицательный тренд не может быть заявлен без хотя
      бы одного события или источника» — сейчас это верно только по
      построению генератора, нет явного правила в валидаторе, которое ловило
      бы нарушение.
- [+] Раздел 14.2/23 Этап 2 критерий: «валидатор отклоняет реальные
      названия» — проверка в коде валидатора есть, но нет прямого
      юнит-теста именно на этот путь (только на генератор, который и так их
      не создаёт).

      проверка всех строковых значений входной конфигурации на признаки
