# Чек-лист задачи: конвейер синтетических данных — эргономика CLI
# (11 небольших улучшений поверх Этапов 1-6, перед Этапом 7)

Источник: собственный аудит `scripts/synthetic_data` по запросу владельца
(не связан напрямую с одним разделом техзадания; ссылки на разделы даны
по месту). Все пункты — аддитивные, не меняющие поведение по умолчанию
там, где это возможно (профиль `balanced` и все существующие флаги должны
остаться такими же).

Ветка: `feat/synthetic-data-cli-ergonomics-v1`.

## Состав работ (11 пунктов)

- [ ] 1. Ленивые импорты в `cli.py`: `validate`/`plan` не тянут
      reportlab/docx/openpyxl/psycopg. `ALL_DOCUMENT_FORMATS` вынесен в
      лёгкий модуль без тяжёлых зависимостей.
- [ ] 2. `--dry-run` для `load-sql`/`cleanup-sql`: печатает цель (хост/БД,
      путь к файлу), не подключаясь; учитывает production-guard в превью.
- [ ] 3. Команда `list`: обход `--output-root`, вывод dataset_id/seed/
      profile/generated_on/размер по каждому найденному датасету.
- [ ] 4. Команда `package --dataset <id>`: пересобирает manifest+ZIP из уже
      отрендеренных файлов на диске, не запуская рендеринг заново.
- [ ] 5. Тест-инвариант: запрет `import requests/httpx/urllib.request/
      socket/aiohttp` во всём `scripts/synthetic_data` (кроме обоснованного
      `psycopg` для явной, подтверждаемой SQL-загрузки).
- [ ] 6. Команда `prune --keep-last N [--confirm] [--dry-run]`: удаляет
      старые датасеты из `--output-root`, оставляя N новейших.
- [ ] 7. Реальная дифференциация профилей (без структурных изменений
      модели): `moderation` — больше комментариев; `data_quality` — ниже
      `data_confidence`/увереннность источника; `crisis` — почти всегда
      отрицательное направление события (кроме архетипа
      `recovering_country`, который остаётся `improved`); `optimistic` —
      метрики смещены к верхней границе диапазона архетипа, инвариант
      «нет идеальной страны» не нарушается; `balanced` — поведение не
      меняется (регресс-тест на неизменность).
- [ ] 8. Автопоиск датасета для `render`/`load-sql`/`cleanup-sql`/`package`:
      если не найден по `--output-root`, ищем по `DEFAULT_OUTPUT_DATA_ROOT`
      и выводим список реально существующих id при неудаче.
- [ ] 9. `scripts/synthetic_data/README.md` — краткий quickstart (~15-20
      строк): генерация, просмотр, загрузка в локальную БД, очистка.
- [ ] 10. Команда `schema`: печатает
      `SyntheticWorld.model_json_schema()` в stdout.
- [ ] 11. Флаги `--json`/`--quiet` для всех world-команд: `--json` — один
      структурированный JSON-объект на stdout вместо текстовых строк;
      `--quiet` — подавляет обычный текстовый вывод (ошибки всё равно в
      stderr).
