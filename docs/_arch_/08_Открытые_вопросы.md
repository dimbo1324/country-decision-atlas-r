## 1. Принятые решения (2026-07-04)

Сроки — не определены (не блокер). Кандидаты-провайдеры для изучения в интеграционном транше: dLocal (Монтевидео), Mercado Pago, PayPal |
| Р-4 | Лицензия контрибуций | Вариант (в): данные стран — неисключительная бессрочная лицензия платформе; авторские методологии — выбор автора при публикации (лицензия платформе \| CC BY-SA) |
| Р-5 | Research-папки `03–07` | Сохраняются, структура не трогается |
| Р-6 | Email-канал | Бэклог; ранняя аудитория покрывается Telegram |
| Р-7 | Названия метрик | Нейминг-ревизия при визуальном транше; стиль — **образный** |

| Р-10 | Период автономной разработки | До переезда владельца в Уругвай (~3–4 месяца от 2026-07): **никаких реальных внешних подключений** (LLM-API, боевой Telegram, почта, платежи, OAuth). Везде, где нужно подключение, уже сейчас закладывается шов |
| Р-11 | E2E/смоук-фикстуры vs. демо-набор (P1-9, Аудит-эпизод 8) | **Вариант Б (интерим), Вариант А — после переключения E2E на синтетический пайплайн.** Сейчас: `--visible` по-прежнему временно снимает `is_demo` только в CI/dev-джобах, зафиксировано и протестировано, что дефолтный путь скрипта демо-набор не расшатывает. `utils/synthetic_data/` (точка входа `scripts/synthetic_data.py`, перенесено из `scripts/synthetic_data`/`docs/synthetic_data` — см. `SYNTHETIC_DATA_PLAN.md` Этап 0) дошёл до Этапа 7 (изолированные `dataset_id`-fixtures и quality-gate интеграция реализованы), но сам переключатель E2E/смоуков на синтетический набор и удаление `--visible` из CI — отдельная, ещё не начатая задача (см. `SYNTHETIC_DATA_PLAN.md` Этап 3). Детали — план, Эпизод 8; `utils/synthetic_data/README.md` |
| Р-12 | i18n-стратегия для UI-контента редизайна (зафиксировано 2026-07-18, Этап 5 «Закрепление») | **Контент ru-only для лейблов интерфейса; `next-intl` остаётся только для «каркаса» приложения.** Не новое решение владельца, а явная фиксация паттерна, который уже сложился де-факто во всех волнах редизайна (Этапы 0-3) и прямо следует из существующего правила `.ai/project/12-domain-rules.md` («не использовать AI-перевод без явного запроса владельца»). Проверено по коду: каталог сообщений `next-intl` (`apps/web/src/messages/{en,ru}.json`, 90 ключей) покрывает только шапку/футер/формы авторизации/палитру поиска/страницы ошибок — ни одного ключа для контента фич или страниц. Все лейблы, добавленные в редизайне (названия табов досье, шаги мастера подбора, лейблы чип-фильтров, подписи каталога) — захардкоженные русские строки, не зависящие от `/en/` vs `/ru/` URL-локали. Следствие: пользователь с `/en/`-локалью видит переведённые навигацию/футер/данные страны (через `overlay_localized_fields` бэкенда), но русские названия табов/шагов мастера — известный, принятый компромисс, не баг. Полная миграция всех этих строк в `next-intl` — явное «не сейчас»: это инвазивный проход по десяткам компонентов ради контента, для которого никто явно не просил английской версии; пересмотреть, если владелец потребует полного EN-паритета интерфейса. |

## 2. Ещё открытые вопросы

### В-1. Первые модераторы

**Статус:** владелец затрудняется ответить (2026-07-04). **Что нужно:** число модераторов фазы 1 и наличие кандидатов; подтверждение/корректировка порогов допуска (репутация, стаж). **Блокер для:** фазы 1 института модераторов — механизм (Эпизод 3) реализуется независимо; вопрос станет актуален к запуску авторского контура (Эпизод 4, invite-only авторы). До этого владелец закрывает модерацию сам.

### В-2. Хостинг и продакшен-развёртывание

**Статус:** владелец затрудняется ответить (2026-07-04). **Что нужно:** площадка (VPS/облако), регион, бюджет; рекомендация остаётся — один хост с docker-compose на старте достаточен. **Блокер для:** публичного запуска Эпизода 2 (напоминания требуют always-on). Разработку не блокирует: fake-режим доставки полноценно тестируется локально. Естественная точка решения — интеграционный транш; для Уругвая учесть латенси до аудитории (вероятно, ЕС/US-East региона достаточно).

### В-3. Сроки монетизации

**Статус:** не определены (2026-07-04), сознательно. **Блокер:** нет — фундамент (Р-3, Р-4, предохранители в реестре инвариантов §6) закладывается независимо от сроков.
