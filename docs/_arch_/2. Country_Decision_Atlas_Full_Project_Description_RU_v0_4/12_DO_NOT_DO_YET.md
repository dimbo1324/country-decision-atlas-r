# 12. What Not To Do Yet

## Не добавлять много стран

Пока нельзя расширяться на 20–50 стран, потому что:

- качество данных упадёт;
- источники сложнее поддерживать;
- legal signals потребуют редакторской дисциплины;
- frontend ещё будет меняться;
- decision engine нужно проверить на малом наборе.

## Не делать AI слишком рано

AI должен быть только после:

- источников;
- evidence;
- data quality;
- content workflow;
- stable API.

Иначе он будет создавать иллюзию знания.

## Не делать форум

Обычный форум рано:

- спам;
- субъективность;
- модерация;
- хаос;
- устаревшая информация.

Лучше сначала structured user stories.

## Не делать marketplace

Marketplace экспертов нужно делать сильно позже, когда появится:

- трафик;
- доверие;
- user intent;
- модерация;
- юридическая модель;
- проверка экспертов.

## Не делать mobile app

Сначала web.

Mobile можно позже, когда:

- продуктовая логика доказана;
- есть retention;
- есть пользователи;
- понятно, какие mobile сценарии нужны.

## Не делать production-perfect backend

Не нужно бесконечно шлифовать backend до идеала перед frontend.

Правильная стратегия:

- backend slice;
- frontend slice;
- backend expansion;
- frontend expansion.

## Не делать сложную авторизацию раньше времени

Временный admin token допустим для MVP. Production RBAC позже.

## Не делать платёжную систему

Пока рано. Сначала надо доказать value.

## Не делать 15–20 языков прямо сейчас

Нужно заложить архитектуру, но реально использовать:

- English;
- Russian.

## Не делать финальный дизайн

Сначала рабочий MVP, потом UX/UI polish.
