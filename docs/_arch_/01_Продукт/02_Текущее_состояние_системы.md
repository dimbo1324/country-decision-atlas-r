# Текущее состояние системы (базовая линия)

> Технический разрез реализованной системы по факту кода (34 роутера, 64 сервиса, 47 репозиториев, 45 миграций, ~150 тестовых файлов / ~1.8 тыс. тестов). Это отправная точка новой линии разработки: всё описанное здесь существует, работает и покрыто тестами. Дата фиксации: 2026-07-04.

## 2. Слои и конвенции кода

| Слой | Где | Правило |
|---|---|---|
| Роутеры | `apps/api/app/api/v1/*.py` | Тонкие: парсинг, зависимости (auth/RBAC/connection), вызов сервиса |
| Сервисы | `apps/api/app/services/` | Вся бизнес-логика; крупные домены — пакетами (`migration_board/`, `decision_engine/`, `data_quality/`) с re-export публичной поверхности через `__init__.py` |
| Репозитории | `apps/api/app/repositories/` | Только SQL (закреплено тестами «repository is SQL-only») |
| Схемы | `apps/api/app/schemas/` | Pydantic v2; OpenAPI-контракт → генерация TS-типов (`pnpm contracts:generate`) |
| Ядро | `apps/api/app/core/` | Settings; database-хелперы; `api_error` → единый формат ошибок `{detail:{error:{code,message,details}}}`; CurrentUser; require_roles |

Конвенции (детально — `CLAUDE.md` в корне):

- **Локализация — overlay на чтении**: `overlay_localized_fields(...)` накладывает переводы поверх строк read-модели; клиент получает статусы `source|translated|fallback|missing`. Легаси-колонки `*_ru/*_en` сохраняются.
- **Пакетная декомпозиция**: файлы >800 строк режутся на пакет; кросс-катящие приватные хелперы — в `helpers.py` с квалифицированным доступом (`helpers.fn()`), иначе monkeypatch в тестах молча перестаёт перехватывать вызовы.
- **Feature flags**: `ensure_feature_enabled(connection, key, message)` → 403 `feature_disabled`; статусы enabled/disabled/internal/deprecated, тиры доступа public/beta/internal/admin, правила по тирам.
- **Тесты**: MagicMock-connection, monkeypatch на модуль репозитория, `_assert_error` для кодов ошибок; migration-тесты читают SQL; каждый файл — однострочный docstring о предмете проверки.

## 3. Карта доменов

### 3.1 Контент-ядро (эталонные данные, publication lifecycle, audit)

| Домен | Ключевые таблицы | API | Особенности |
|---|---|---|---|
| Страны и карточки | countries, country_profiles, country_cards | `/countries`, `/countries/{slug}`, `…/card` | **Тестовый набор: RU/UY/AR**; RU/EN; lifecycle draft→review→published |
| Источники и доказательства | sources, evidence_items | `/sources`, `…/evidence` | Тип/язык/confidence; наружу — только published |
| Правовые сигналы | legal_signals, legal_signal_events | `/legal-signals`, таймлайн | legal_status, impact direction/level, таймлайн по годам |
| Маршруты | routes (+documents/sources/evidence), route_checklist_items | `/routes/*` | Шаг чек-листа: документы, стоимость, сроки, официальные требования — с источниками |
| Пары стран | country_pair_compatibility (+sources/evidence) | `/country-pairs/*` | origin→destination: визы/налоги/банкинг/перелёты/язык/ограничения |
| Справочное | glossary, methodology_sections, data journal | `/glossary`, `/methodology`, `/data-journal` | |

### 3.2 Решения

| Возможность | Механика |
|---|---|
| `/decision/run` | 5 сценариев × 7 критериев; сохранённые breakdowns; ранжирование `(-score, -confidence, slug)`; strengths ≥70 / weaknesses ≤50; risk warnings; всё с source_ids |
| Кастомные веса | per-request; валидация полноты/диапазона/суммы; пересчёт runtime без записи |
| Persona | Слой: модификаторы метрик (−0.5…+0.5) → скорректированные веса (сумма = 1) → persona-adjusted ранжирование |
| Origin-aware | pair-контексты в результатах при указании страны происхождения |
| Сравнение | winner/tie: дельта <3 — tie/low, <10 — medium, иначе high |
| Паспорта | Снапшот запроса и результата; публичный токен (в БД hash+prefix), TTL, 410 на истёкший |
| Wizard | Цель → сценарий; слайдеры → веса |

### 3.3 Вычисляемые слои (производные, версионируемые)

| Слой | Хранение | Пересчёт |
|---|---|---|
| CII | cii_metric_definitions (полярность, источник, active), country_metric_values, scenario_metric_weights (**version**), country_cii_scores (formula_version, geometric) | сид + admin recompute |
| Platform metrics (LVI, SSRS, Contradiction) | country_platform_metrics (methodology_version) | скрипт, идемпотентно, dry-run |
| Trust Score | country_trust_scores (label, confidence, components, expires_at) | скрипт; из counts/freshness/contradiction |
| Drift | country_drift_snapshots (period, label, confidence) | из legal_signal_events; downgrade при малых данных |

### 3.4 Платформа и пользователь

- **Auth/RBAC**: PBKDF2-SHA256 (260k итераций); сессии с TTL и revoke-all; роли user/editor/moderator/owner; промоция в owner — только owner'ом; регистрация за флагом; Telegram-линковка через gRPC к нотификатору.
- **Watchlists** (за флагом), **What changed** (окно/`since` по стране), **Search** (search_documents + GIN, rebuild-скрипт, локали раздельно), **Analytics** (hash сессий, санитизация metadata, запрещённые ключи), **Feature flags API**, **Cache**.

### 3.5 Сообщество

- **Q&A**: pending→published (модерация); консенсус прозрачной формулой (взвешивание + бонус source-backed + детект «спорного» по разбросу).
- **User stories**: структурированные, verification status, маркировка synthetic; рейтинги по 6 осям; репорты ошибок данных.
- **Миграционная доска**: посты с lifecycle и модерацией (approve/reject/hide), видимость public/members_only/private, PII-детектор публичных текстов (email/телефон/@handle/URL), контакт-запросы по взаимному согласию (лимит 20/день), блокировки, репорты (20/день, дедупликация), companion matching с причинами совпадения, публичная проекция без PII.

### 3.6 AI-контур (инструментальный)

Провайдерный шов (Fake-by-default); контекст — только published-контент и сохранённые метрики; `/ai/ask` (цитаты, отказ без контекста), `/ai/explain-number`, `/ai/decision-intent`; драфты → needs_review; contradiction candidates → review; interaction logs с ограничениями metadata.

### 3.7 Редакция и операции

Admin CRUD с lifecycle и audit_events; **data quality** — отчёт valid/invalid, 60+ проверок пакетом по доменам; translation jobs + worker foundation; outbox relay (батчи, max attempts, метрики); rate limits.

## 4. Операционный контур

- **Миграции**: `database/migrations/NNN_name.sql`; `schema_migrations(version=имя файла, checksum)` — применённые файлы неприкосновенны.
- **Quality gate**: `python dev_tools_scripts_runner.py [--profile …] [--doctor] [--fix]` → `scripts/dev_tools/full_check.py`: диагностика окружения, тулчейн, зависимости, статика (ruff/mypy/sqlfluff), pytest, Go vet/test, Docker-стек + миграции + runtime-смоки + E2E, pre-commit; отчёты в `full-check-reports/`.
- **CI**: `.github/workflows/quality.yml`.
- **Периодические джобы** (пока запускаются вручную/через `dev_tools_scripts_runner.py`, не подключены к cron/systemd/CronJob — деплой-цель ещё не выбрана):
  - `scripts/outbox_relay.py` — `domain_events` → Kafka, идемпотентно по `event_key`, батчами.
  - `scripts/dispatch_trip_reminders.py` (алиас `dispatch-trip-reminders` в оркестраторе) — переносит просроченные `trip_reminders` в `domain_events` (`event_type=trip_reminder_due`), транзакционно, `SKIP LOCKED`, идемпотентно. До появления реального деплоя запускать вручную; на боевом окружении — повесить на регулярное расписание.
