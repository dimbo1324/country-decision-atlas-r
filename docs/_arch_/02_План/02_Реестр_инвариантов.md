# Реестр архитектурных инвариантов

> Решения, которые нельзя нарушить ни в одной будущей задаче. Нарушение любого пункта — блокер merge независимо от зелёного quality gate. Изменение самого реестра — только выделенной документационной задачей по решению владельца.

## 1. Ядро и данные

1. **PostgreSQL — единственный source of truth.** Go/Kafka/Mongo — только асинхронная доставка и производное, перестраиваемое состояние.
2. **Математика ядра неизменна**: формула CII, scenario-веса, scoring decision engine не редактируются. Всё новое — слоями поверх (Persona, Drift, Trust, профили весов, авторские слои).
3. **Производные метрики** (LVI, SSRS, Contradiction, Trust, репутации) считаются из собственных данных и **никогда не примешиваются** в агрегированный CII.
4. **Методология платформы меняется только новой версией** (`methodology_parameters.version`, `formula_version`): никаких «тихих» правок значений; каждая расчётная выдача и паспорт несут версию.
5. **Source-backed everything**: публичное утверждение без источника/атрибуции не публикуется; data-quality гейты не ослабляются и не обходятся фейковыми данными.
6. **Жизненный цикл контента**: `draft → review → published → archived` (+ `rejected`); переходы — с audit-событием.
7. **Миграции**: последовательные, идемпотентные, чистые по sqlfluff; применённые файлы не переименовываются и не редактируются (checksum-трекинг `schema_migrations`).
8. **Демо-набор стран не удаляется** (решение Р-1): после консервации (Эпизод 5) демо-данные скрыты с публичных поверхностей, но сохранены в фикстурах и восстановимы dev-инструментом идемпотентно.

## 2. Асинхронный контур

9. **Outbox-дисциплина**: события — только через `domain_events`; `event_key` обеспечивает идемпотентность; событие публикации — на переходе в `published`; seed/историческое — `notifiable = FALSE` (без шторма уведомлений).
10. **Notifier**: абстракция канала + DLQ; Mongo — eventually-consistent производное, перестраиваемое из Kafka-лога.

## 3. Внешние системы

11. **Провайдерные швы, Fake-by-default**: нейросеть, Telegram, будущий платёжный провайдер подключаются через абстракцию с fake-реализацией по умолчанию; переключение — настройкой (`*_MODE=fake|real`), без переписывания кода.
12. **AI — инструмент на поводке**: отвечает только из published-контента с цитатами; без контекста — отказ; AI-контент попадает к пользователям только через human review; AI не ранжирует ответы людей.
13. **Автономный режим разработки** (решение Р-10, до интеграционного транша): real-mode внешних сервисов не включается; каждый эпизод обязан быть полностью реализуемым и тестируемым офлайн. Если боевое подключение потребует дописывать логику — шов спроектирован неверно, это дефект эпизода.

## 4. Права, приватность, безопасность

14. **Deny by default; проверка прав только на сервере**; каждое привилегированное действие — в audit_events с причиной.
15. **Флаги ≠ права**: feature-флаги управляют раскаткой функциональности, гранты/роли — допуском пользователя.
16. **Репутация — не право**: производная репутация информирует, но никогда автоматически не выдаёт способности; выдача гранта — явное действие governance.
17. **Никогда не храним**: проездные документы, брони, сканы документов; сырые share-токены (только hash+prefix); IP/user-agent в аналитике.
18. **Private-first**: личные объекты создаются приватными; публикация — явное действие; публичные проекции проходят PII-фильтр.
19. **k-анонимность агрегатов о людях**: группы меньше порога (параметр `flows.k_anonymity`, стартовое значение 20) не раскрываются ни при каких параметрах API.
20. **Доступ модератора к приватному** — только в контексте поданного репорта и только к его предмету; каждый такой доступ аудируется.
21. **«Не юридическая консультация»** и дата последней проверки — на всём публичном, авторском и доставляемом контенте.

## 5. Авторский и пользовательский контент

22. **Сепарация слоёв**: авторские метрики/методологии никогда не смешиваются с платформенными в одной цифре — только рядом, с явной атрибуцией «автор, версия, обновлено».
23. **Методология обязательна**: авторская метрика без описания «как посчитано» не публикуется; каждое значение — источник ИЛИ явная пометка «личный опыт автора».
24. **Форк несёт родословную** (`forked_from`), атрибуция контрибуций сохраняется.
25. **Контрибуция стран** — только через onboarding-гейт (автоматический стандарт готовности) и куратора-редактора; публикация страны без куратора невозможна.
26. **Синтетический контент** всегда маркирован и вытесняется реальным по мере покрытия.
27. **Лицензии контрибуций** (решение Р-4): данные стран — неисключительная бессрочная лицензия платформе; авторские методологии — выбор автора при публикации (платформенная лицензия | CC BY-SA), выбор хранится с определением метрики.

## 6. Монетизация (предохранители до реализации)

28. **Деньги не покупают** права, репутацию, позиции в выдаче или прохождение модерации.
29. Платёжная интеграция — только через провайдерный шов (п. 11) и только в интеграционном транше; платёжные данные не хранятся на платформе. Юрисдикция оперирования — Уругвай (решение Р-3).

## 7. Стек

30. **Ядро — FastAPI + PostgreSQL**; «Postgres до триггера»: тяжёлые вычисления выносятся из Python только при доказанном узком месте, заменой изолированного модуля за швом.
